# HarmonyOS 设计指南 — 完整参考

> 来源：华为开发者联盟 HarmonyOS 设计指南
> 更新日期：2025-06-20

---

## 1. 设计理念 — One Harmonious Universe

| 层级 | 理念 | 要点 |
|------|------|------|
| **One** | 万物归一，以人为本 | 人因研究驱动设计，文字清晰易读、图标精确、色彩舒适、动效流畅；交互直觉化，操作符合本能 |
| **Harmonious** | 平衡共生 | 物理世界映射数字界面（光影/材质），虚实融合降低认知成本 |
| **Universe** | 演化自如 | 多设备自适应，一致性（减少学习成本）与差异性（因设备而异）平衡；数字健康/全球化/无障碍探索 |

## 2. 视觉风格

### 2.1 色彩

**Token 化色彩系统**，分层构建：

| 层级 | 说明 |
|------|------|
| 品牌色 (Brand) | 体现品牌特征的主色 |
| 语义色 (Semantic) | 成功/警告/错误/信息等状态色 |
| 文本色与图标色 (Text & Icon) | 主/次/禁用层级 |
| 组件容器色 (Container) | 卡片、按钮等背景 |
| 界面背景色 (Background) | 页底/页面的层级背景 |

### 2.2 字体 — HarmonyOS Sans（鸿蒙黑体）

- 支持：简繁中文、Latin、Greek、Cyrillic、Arabic
- 设计理念：现代屏显与人文笔意之间寻找平衡
- 无衬线字体，专为 UI 显示优化

### 2.3 应用图标

- 风格：轻拟物 + 半透明模糊
- 网格布局，由上至下光照渐变色
- 输出规范：1024×1024px，PNG 分层资源
- 适配范围：手机/折叠屏/平板/PC/智慧屏/穿戴

### 2.4 HarmonyOS Symbol（系统图标）

- 与文字基线对齐，支持字重变化
- 三种颜色渲染策略：`SINGLE` / `MULTIPLE_OPACITY` / `MULTIPLE_COLOR`
- 7 类动效策略：Appear / Disappear / Bounce / Scale / Replace / Pulse / Hierarchical
- 几何网格设计，角标规则和裁剪规则统一

## 3. 布局

### 3.1 布局基础

| 概念 | 说明 |
|------|------|
| **vp**（虚拟像素） | 跨屏幕密度一致尺寸 |
| **fp**（字体像素） | 默认 = vp，跟随用户字体大小设置缩放 |
| **8vp 网格** | 基本对齐/间距单位 |
| **断点系统** | 横屏断点：0-600 / 600-840 / 840+ vp；竖屏断点：按宽高比 |
| **栅格系统** | 0-600vp → 4列；600-840vp → 8列；840+vp → 12列；最大可用宽 2220vp |
| **系统安全区** | 避开挖孔/圆角/状态栏/导航栏/折叠铰链 |

#### HarmonyOS 设备屏幕尺寸

| 类型 | 机型 | 尺寸 (vp) |
|------|------|-----------|
| 手机 | Mate 60/Mate 70/Pura 70 系列 | ~393vp |
| 折叠内屏 | Mate X5 / Mate XT | ~756vp / ~848vp |
| 阔折机 | Pura X | ~390vp |
| 平板 | MatePad Pro 系列 | ~840-1000vp |
| PC | MateBook Pro / MateBook Fold | ~1280-1512vp |
| 穿戴 | 圆形 466×466 / 方形 408×480 | -- |

### 3.2 响应式应用架构

| 导航模式 | 断点条件 | 说明 |
|----------|----------|------|
| 侧边 Tab | ≥ 840vp | 底部 Tab 挪移到左侧 |
| 分栏 (Split) | ≥ 600vp | 父子层级平行显示，默认 4:6，可配 5:5 或 6:4 |
| 侧边导航栏 (Sidebar) | ≥ 840vp | 支持嵌入/悬浮两种显示方式，可临时收起 |

### 3.3 响应式布局方法（8 种）

| 方法 | 适用场景 |
|------|----------|
| 自适应拉伸 | 左右拉伸 / 均分拉伸 — 文本、按钮、间距 |
| 自适应缩放 | 等比缩放 / 占比缩放 — 图片、圆形按钮、Banner |
| 自适应延伸 | 组件按优先级扩展/隐藏 — Tab、操作块 |
| 自适应折行 | 内容自动换行 — 操作块、内容流 |
| 缩进布局 | 宽屏不同缩进效果 — 8/12列下使用6/8列缩进 |
| 挪移布局 | 宽高比变化 >200% 时纵横切换 |
| 重复布局 | 水平排列同级组件 |
| 分栏布局 | ≥ 600vp 并排显示层次内容 |

## 4. 动效 — "引力"动效体系

### 4.1 设计原则

| 原则 | 说明 |
|------|------|
| 自然流畅 | 帧率稳定、触控跟随、跟手操作 |
| 简洁高效 | 最小运动、清晰路径 |
| 快速响应 | 短距 ≤ 250ms，长距 ≤ 350ms |
| 运动一致 | 运动路径统一 |
| 隐藏等待 | 通过动效减少等待焦虑 |
| 凸显品牌 | 拖尾/熔球/辉光/磁吸等品牌特效 |

### 4.2 动效属性

| 属性 | 说明 |
|------|------|
| **时长 (Duration)** | 单位 ms，根据复杂度和面积范围确定 |
| **曲线 (Curve)** | 贝塞尔曲线（标准/减速/加速）；弹性曲线（interpolatingSpring / springMotion / responsiveSpringMotion） |
| **帧率 (Frame Rate)** | 交互动效帧率 vs 纯动效帧率 |
| **弹性曲线特点** | 可中断、速度继承、自然定时的特点 |
| **阻尼类型** | 临界阻尼 / 欠阻尼 / 过阻尼 |

### 4.3 转场动效

| 类型 | 说明 |
|------|------|
| 同层跳转 | 平级页面切换 |
| 上下层跳转 | 进入/返回的层级动效 |
| 跨层跳转 | 跨层级导航 |
| 一镜到底 | 共享元素 / 共享容器 / 共享动量 |
| 淡入淡出 | 单向淡入 / 交叉淡入淡出 |

**LTR vs RTL**：层级跳转动效在 RTL（阿语）语言下方向相反。

### 4.4 手势动效

| 手势 | 要点 |
|------|------|
| 点击 (Click) | 平均按压 100-130ms，按下即触发反馈 |
| 滑动 (Slide) | 跟手 → 离手瞬间 → 离手后，连贯性关键 |
| 翻动 (Flip) | fling + drag 操作，snap-to-limit 属性，成功/失败态 |
| 夹捏 (Pinch) | 多指缩放/旋转/移动，跟手 → 离手 → 离手后三阶段 |
| 拖拽 (Drag) | 标题联动（顶部边界阻尼）、卡片左右滑（阻尼引导）、下拉返回（渐进阻尼 + 速度继承） |

## 5. 界面用语

### 5.1 写作原则

1. **友好尊重** — 使用"您"，避免正式机械
2. **必要信息** — 少谈卖点，多谈买点；重要信息先说
3. **简洁短句** — 删减多余字词
4. **清晰易懂** — 避免生僻/技术化词语
5. **鼓励正向** — 少用"失败""错误"，用"只需""简单""轻松"
6. **逻辑统一** — 用词一致、风格一致、视角一致

### 5.2 语气语调

整体像"知识渊博的朋友"，分场景：

| 场景 | 语气 | 示例 |
|------|------|------|
| 新功能引导 | 轻松 | "尽情发现服务卡片，无需安装，即用即走" |
| 普通提示 | 平和 | "您可通过悬浮窗或分屏的方式同时使用多个应用" |
| 空状态/等待 | 安抚 | "您还没有播放记录，快去看看精彩内容吧" |
| 重要授权 | 严肃 | "删除后将无法恢复。确定删除？" |

### 5.3 典型场景模板

#### 删除
- **无大影响**：`"是否删除 X 个XXX？"`
- **有大影响**：`"删除后，XXX。确定删除？"`

#### 异常提示
- **网络异常**：`"服务器忙，请稍后再试"` / `"网络未连接，请检查网络设置"`
- **登录异常**：`"用户名或密码错误，请重新输入"` / `"密码错误。还可输入 X 次"`
- **失败提示**：`"XXX失败，请XXX"` / `"XXX，XXX失败"`
- **超限异常**：`"XXX已达上限"`

#### 内容加载
- `"正在加载，请稍候…"` / `"正在发送，请稍候…"`

#### 检查更新
- `"有新版本"` / `"已是最新版本"` / `"立即更新"` / `"WLAN 下自动更新"`

### 5.4 编辑规范

| 标点 | 规则 |
|------|------|
| **句号** | 短语列举/网址/设置二级文本/Toast 不用句号；操作步骤/句子列举用句号 |
| **顿号** | 并列词或词组之间 |
| **分号** | 并列句子之间 |
| **感叹号** | 尽量少用，仅在警告/恭喜场景 |
| **问号** | 仅表达询问 |
| **冒号** | 由"以下""如下"引出列举时 |
| **引号** | 应用名/功能名/按键名加引号；图标不加 |
| **省略号** | 用半角三点 `...` |
| **连接号** | 短横线 `-` 无空格（编号/号码）；波浪线 `~` 无空格（数值范围） |
| **操作路径** | `前往"设置">"应用">"应用管理"` |

| 空格 | 规则 |
|------|------|
| **加空格** | 英文字符前后 / 数字前后 / 度量单位前 / © 前后 / `+` 号前后 / 序号后 |
| **不加空格** | `%` 与数值间 / 数学符号与数值间 / 货币符号后 / `-` 前后 / `>` 前后 / `/` 前后 |

| 词语 | 规范 |
|------|------|
| **您** | 统一使用，避免连续出现 |
| **数词** | 统一阿拉伯数字，数字前后半角空格 |
| **请** | 具体行为用"请"，列举时不宜连续 |
| **的/地/得** | 正确区分；标题不宜出现"的" |
| **英文词语** | 首字母大写，与中文间加半角空格（如"App"非"APP"） |

## 6. 全球化（i18n + L10n）

### 6.1 核心原则

- **国际化 (i18n)**：一套方案满足不同国家
- **本地化 (L10n)**：针对各国的个性化方案

### 6.2 视觉

| 维度 | 规范 |
|------|------|
| **文字与图片** | 不要将文字嵌入图片，分层展示 |
| **颜色** | 避免各国禁忌色（俄罗斯黄色=分手、巴西/英国紫色=悲伤/不吉利） |
| **图标** | 避免宗教/政治/种族敏感元素；RTL 语言需提供方向镜像资源；图标上尽量不加文字 |
| **插画** | 中性风格，不太暴露/个性化/地域化；RTL 需提供镜像插画 |
| **Emoji** | 积极阳光，避免色情暴力、手势、国旗代表语言、宗教神话符号 |

### 6.3 布局

- 以英文为基准预留空间：≤10字符预留100%、10-20字符预留50%、>20字符预留30%
- RTL（阿拉伯语等）事件发展顺序从右到左

### 6.4 动效与音效

- LTR：下一层从右向左滑入；RTL 方向相反
- 音效不能出现宗教/政治敏感音轨

### 6.5 界面用语

- 字体少用粗体/斜体/下划线（超链接除外）
- 海外默认语言为英文，基于英文设计
- 标点转换为本地方案

### 6.6 其他规范

| 维度 | 标准 |
|------|------|
| 排序 | 不仅考虑 A-Z，还需考虑多音字/变音符号 |
| 日期/数字/货币 | 基于 ISO/CLDR/UTC 标准 |
| 每周首日 | ISO 标准周一（亚欧）/ 北美周日 / 伊斯兰周六 |
| 长度/温度 | 除美国/利比里亚/缅甸外均用公制/摄氏度 |
| 货币 | 调用标准接口配合本地语言 |
| 知识产权 | 验证所有商标声明和当地许可 |

## 7. 隐私

### 7.1 授权方式

| 方式 | 说明 |
|------|------|
| **按钮式授权控件** | 系统预置临时授权，无需弹框，用户主动触发获取数据 |
| **选择式授权控件** | 临时选择图库/文件/相机/联系人，无需弹框 |
| **弹框授权** | 长期/后台获取数据时使用，不允许连续两个以上弹框，不允许提前申请 |

### 7.2 权限使用说明撰写

| 标准 | 说明 |
|------|------|
| 清晰准确 | 描述权限允许访问后的功能 |
| 简洁易懂 | 避免技术化/专业术语 |
| 正例 | "用于扫码拍照和添加好友" |
| 反例 | "用于使用摄像头" |

### 7.3 用户不同意授权

- 不允许应用闪退/强制退出/重复弹框
- 应引导用户前往"设置"手动开启
- 再次触发场景时提供开启引导
